Interception 기반 키보드/마우스/픽셀 매크로 편집기의 주요 기능을 처음 사용하는 기준으로 정리한 문서입니다.

1. 프로그램 개요

Macro는 여러 매크로를 프로필 파일로 관리하고, 각 매크로 안에 액션을 트리 형태로 배치해서 실행하는 GUI 프로그램입니다. 키 입력, 마우스 클릭/이동, 대기, 조건 분기, 반복, 픽셀 검사, 변수 저장, 타이머, 텔레그램 메시지까지 한 흐름 안에서 구성할 수 있습니다.

프로필여러 매크로와 전역 변수, 입력장치 설정을 저장하는 파일입니다.
매크로트리거 키/마우스와 실행 모드를 가진 실행 단위입니다.
액션키 입력, 마우스 클릭, 조건, 반복 같은 실제 동작입니다.
조건키, 픽셀, 변수, 타이머, 시간/날짜 상태를 검사합니다.
Windows에서 관리자 권한으로 실행해야 키보드/마우스 후킹과 Interception 기반 입력 기능이 정상 동작합니다.

2. 실행과 기본 흐름

설치

프로젝트 폴더에서 의존성을 설치합니다.

uv sync

실행

관리자 권한 터미널에서 실행합니다.

uv run python main.py

처음 사용하는 순서

  1. 설정 > 키보드 설정에서 입력 모드와 장치를 확인합니다.
  2. 매크로 추가로 새 매크로를 만들고 트리거를 지정합니다.
  3. 액션 트리에 액션 추가를 눌러 필요한 동작을 순서대로 넣습니다.
  4. 적용을 눌러 GUI 내용을 엔진에 반영합니다.
  5. 시작과 활성 상태를 켜고 트리거로 테스트합니다.
  6. 문제가 있으면 로그, 픽셀 테스트, 디버거로 확인합니다.

3. 메인 화면

상단 상태/제어 영역

항목설명
시작매크로 엔진을 시작합니다. 시작 후에도 활성 상태가 꺼져 있으면 트리거 실행은 되지 않습니다.
중지엔진을 멈추고 눌림 유지 중인 입력을 정리합니다.
일시중지실행 중인 매크로 흐름을 잠시 멈춥니다.
활성/비활성엔진은 켜 둔 상태에서 트리거 반응만 켜거나 끕니다.
적용현재 GUI 설정을 엔진에 반영합니다. 단축키는 Ctrl+Shift+P입니다.

메뉴

메뉴기능
파일새로 만들기, 불러오기, 저장, 다른 이름으로 저장. 각각 Ctrl+N, Ctrl+O, Ctrl+S, Ctrl+Shift+S를 사용할 수 있습니다.
기능스크린샷, 디버거, 픽셀 테스트, 픽셀 패턴 관리, 전역 변수 관리, 프리셋 옮기기, 색상값 계산, 이미지 뷰어/피커를 엽니다.
설정키보드/마우스 입력 방식과 Interception 장치를 설정합니다.

매크로 목록

매크로 목록에는 순번, 이름, 트리거, 모드, 활성 여부, 적용 범위, 설명이 표시됩니다. 하단 버튼으로 추가, 편집, 복사, 붙여넣기, 복제, 삭제가 가능합니다.

로그

실행 상태, 픽셀 테스트 결과, 입력장치 테스트 결과 등이 표시됩니다. 필요 없을 때는 로그 표시를 끌 수 있습니다.

4. 프로필 관리

프로필은 현재 매크로 목록, 변수, 입력장치 설정, 픽셀 패턴 참조, 해상도 기준값 등을 저장하는 단위입니다.

  • 새로 만들기: 현재 편집 내용을 초기화하고 새 프로필을 시작합니다.
  • 불러오기: 저장된 JSON 프로필을 엽니다.
  • 저장: 현재 파일에 덮어씁니다.
  • 다른 이름으로 저장: 새 파일로 저장합니다.
  • 최근 불러온/즐겨찾기: 자주 쓰는 프로필을 빠르게 다시 열 수 있습니다. 즐겨찾기는 드래그로 순서를 바꿀 수 있습니다.
프로필을 수정한 뒤에는 저장과 적용을 구분해서 사용하세요. 저장은 파일 기록이고, 적용은 현재 실행 엔진에 반영하는 동작입니다.

5. 매크로 만들기

매크로의 기본 구성

항목설명
이름/설명목록에서 구분하기 위한 표시 이름과 설명입니다.
트리거매크로를 실행할 키 또는 마우스 입력입니다. 예: z, shift+1, mouse1.
모드홀드, 토글, 1회 실행 같은 실행 방식을 정합니다.
활성개별 매크로를 켜거나 끕니다.
적용 범위특정 앱에서만 실행하거나 특정 앱을 제외하는 식으로 범위를 제한할 수 있습니다.
사이클 횟수비워 두면 계속 반복하고, 숫자를 입력하면 지정 횟수만 실행합니다.

액션 트리 편집

매크로 편집 창에서 액션을 추가하고 순서를 바꿉니다. 조건, 그룹, 반복 액션은 하위 액션을 가질 수 있어 트리 구조가 됩니다. IF 액션은 참 분기, ELSE, ELIF를 통해 흐름을 나눌 수 있습니다.

  • 액션 추가: 선택한 위치에 새 액션을 넣습니다.
  • 선택 편집: 선택 액션의 세부 설정을 엽니다.
  • 선택 삭제: 선택한 액션을 제거합니다.
  • 복사/붙여넣기: 자주 쓰는 액션 묶음을 재사용합니다.
  • 드래그앤드롭: 액션 순서를 바꾸거나 하위로 이동합니다.

6. 액션 설정

액션 설정 창의 타입에 따라 필요한 입력칸만 보입니다. 공통으로 이름, 설명, 활성 여부, 반복 횟수, 조건, 입력 딜레이 오버라이드 등을 설정할 수 있습니다.

타입설명주요 입력

마우스 누르기/down마우스 버튼을 누른 상태로 유지합니다.마우스 버튼, 좌표 x,y
마우스 떼기/up눌려 있는 마우스 버튼을 뗍니다.마우스 버튼, 좌표 x,y

시간/흐름 액션

타입설명
대기/sleep지정 시간 동안 기다립니다. 500ms, 3초, 1.5분, 1~3 같은 입력을 사용할 수 있습니다.
라벨/label점프 대상 위치를 만듭니다.
점프/goto지정 라벨로 이동합니다.
continue현재 매크로 사이클 처음으로 돌아갑니다.
return현재 매크로 실행을 종료합니다.
break현재 그룹 또는 while/repeat_n 반복을 빠져나옵니다.
macro_stop현재 매크로를 즉시 중단합니다.
noop아무 동작도 하지 않습니다. 자리 표시나 임시 비활성 용도로 씁니다.

조건/그룹 액션

타입설명

조건/elifIF의 추가 조건 분기로 사용합니다.
그룹/group하위 액션 묶음을 실행합니다. 모두 실행, 첫 참만 실행, 첫 참 후 continue/return, while, repeat_n 모드를 제공합니다.

기능 액션

타입설명
픽셀겟/pixel_get지정 좌표의 색상을 읽어 color 변수에 저장합니다. 좌표는 x,y 또는 x,y,w,h 형식입니다.
변수 설정/set_varValue 변수를 직접 설정하거나 현재값에 더하고 뺍니다. 범위값을 쓰면 실행마다 랜덤값을 사용할 수 있습니다.
타이머/timer타이머1~20 중 하나를 지정 값으로 초기화합니다.
사운드 알림시스템 알림음 또는 선택한 사운드 파일을 재생합니다.
다른 매크로 1사이클 실행선택한 매크로를 한 사이클 실행합니다.
텔레그램 메시지봇 토큰, 채팅 ID, 메시지를 이용해 텔레그램으로 알림을 보냅니다.
컴퓨터 종료운영체제 종료 명령을 실행합니다. 실사용 전 테스트 프로필에서 충분히 확인하세요.

입력 딜레이 오버라이드

키/마우스 입력 액션마다 기본 딜레이와 다른 누름-뗌 지연, 키 사이 지연을 줄 수 있습니다. 40처럼 고정값 또는 40-80처럼 랜덤 범위를 사용할 수 있습니다.

7. 조건 설정

조건은 IF, ELIF, 그룹 모드, 매크로 실행 판단, 디버그 테스트에서 사용됩니다.

조건 타입설명
픽셀화면 영역에서 특정 색상이나 픽셀 패턴이 있는지 검사합니다. 허용오차, 최소 일치 픽셀 수, 있음/없음 판정을 설정합니다.
키/마우스키 또는 마우스 버튼의 press, down, up, hold, released 상태를 검사합니다. 여러 키를 AND/OR로 묶을 수 있습니다.
변수 조건Value 변수의 값이 같은지, 다른지, 짝수인지, 홀수인지 검사합니다.
타이머타이머1~20의 현재 값과 기준 값을 비교합니다. >=, >, ==, <=, <, !=를 사용할 수 있습니다.
시간/날짜매일 시각, 날짜, 날짜+시간을 기준으로 현재 시간과 비교합니다. 범위 입력도 사용할 수 있습니다.
AND 그룹하위 조건이 모두 참일 때 참입니다.
OR 그룹하위 조건 중 하나라도 참이면 참입니다.

키/마우스 조건 모드

  • press, down: 눌리는 순간 한 번 참입니다.
  • up: 뗄 때 한 번 참입니다.
  • hold: 누르고 있는 동안 계속 참입니다.
  • released: 누르지 않은 상태일 때 참입니다.

픽셀 조건 입력

  • 영역: x,y 또는 x,y,w,h. w/h를 생략하면 1x1로 처리합니다.
  • 색상: ff0000 또는 #ff0000 형식입니다.
  • 변수: /name, ${name}, @name 형식으로 참조할 수 있습니다.
  • 패턴 사용: 픽셀 패턴 관리에서 저장한 패턴 이름을 선택합니다.

8. 전역 변수

전역 변수는 여러 매크로와 조건에서 공유됩니다. 메인 화면의 변수 탭 또는 기능 > 전역 변수 관리에서 관리합니다.

종류용도예시
Sleep대기 시간 값500, 1~3, 2초
Region픽셀 검사 영역100,200,50,50
Color픽셀 색상ff0000
Key키 이름 또는 스캔코드space, shift+1
Value일반 값, 숫자, 문자열10, ready

참조 방식

입력칸에서 변수는 /변수명, ${변수명}, @변수명 형태로 참조합니다. 변수 입력이 가능한 칸에서는 /를 입력하면 자동완성이 표시됩니다.

변수 설정 액션

set_var 액션으로 Value 변수를 직접 설정하거나, 현재값에 더하기/빼기를 할 수 있습니다. 증감 모드는 현재값이 숫자로 해석될 수 있어야 합니다.

9. 픽셀/패턴

픽셀 테스트

기능 > 픽셀 테스트에서 영역, 색상, 허용오차, 최소 픽셀 수를 넣고 조건이 실제 화면에서 참인지 확인할 수 있습니다. 조건 편집창의 테스트 버튼에서도 같은 디버그 흐름을 열 수 있습니다.

색상값 계산

기능 > 색상값 계산은 목표 색상과 샘플 색상의 차이를 보고 적절한 허용오차를 결정할 때 사용합니다.

픽셀 패턴 관리

단일 색상 하나가 아니라 여러 픽셀의 상대 좌표와 색상 조합을 검사하려면 패턴을 사용합니다. 패턴은 pattern 폴더의 JSON 파일로 저장되며 여러 프로필에서 공유됩니다.

  • 패턴 이름을 만들고 기준점 주변의 샘플 좌표/색상을 추가합니다.
  • 조건 설정에서 패턴 사용을 켜고 저장된 패턴을 선택합니다.
  • 이미지 뷰어/피커와 디버거를 같이 쓰면 좌표와 색상을 빠르게 확인할 수 있습니다.

10. 이미지 뷰어/피커

기능 > 이미지 뷰어/피커는 스크린샷 폴더나 선택한 루트 폴더의 이미지를 보면서 좌표와 색상을 뽑는 창입니다.

주요 기능

  • 루트 변경, 탐색기에서 열기, 새로고침, 선택 삭제, 폴더 비우기.
  • 파일 트리에서 이미지 선택, 폴더 이동, 드래그앤드롭 이동/복사.
  • 즐겨찾기 그룹 추가, 현재 폴더 추가, 삭제.
  • 단일 캡처 후 자동 새로고침.
  • 이미지 위 마우스 위치의 좌표와 색상 확인.
  • 범위 선택으로 x,y,w,h 영역을 복사하고 조건 창에 자동 반영.

단축키

F2현재 이미지 위 샘플 색상을 복사합니다.
Ctrl+F1범위 선택을 시작합니다.
Ctrl+Left/Right이전/다음 이미지로 이동합니다.
Arrow샘플 위치를 1픽셀씩 이동합니다.
+/-/0확대, 축소, 배율 초기화.
F5새로고침.
Delete선택 삭제.
Esc창 닫기 또는 범위 선택 취소.

11. 스크린샷

기능 > 스크린샷에서 화면 캡처를 시작/중지하고 저장 형식과 단축키를 설정합니다. 저장된 이미지는 이미지 뷰어/피커에서 바로 확인할 수 있습니다.

  • 캡처 간격과 저장 큐 크기 제한을 둡니다.
  • PNG 또는 JPG 형식을 사용할 수 있습니다.
  • JPG 품질, PNG 압축 수준을 설정할 수 있습니다.
  • 시작, 중지, 단일 캡처 핫키를 지정할 수 있습니다.
스크린샷 핫키는 매크로 트리거와 충돌하지 않도록 별도 키를 사용하는 것이 좋습니다.

12. 디버거와 테스트

디버거

기능 > 디버거는 조건 평가 결과를 트리로 보여 줍니다. 픽셀 조건, 변수 조건, 타이머 조건이 왜 참/거짓인지 확인할 때 사용합니다.

  • 평가 주기와 허용오차를 조정할 수 있습니다.
  • 조건 실패 경로와 세부 값을 확인할 수 있습니다.
  • 이미지 뷰어의 현재 이미지를 디버그 소스로 사용할 수 있습니다.
  • 실패 시 캡처 기능으로 조건이 실패하는 순간의 화면을 저장할 수 있습니다.

픽셀 테스트

단일 픽셀/영역 조건만 빠르게 확인하려면 픽셀 테스트 창을 사용합니다. 조건 창에서 테스트를 누르면 현재 조건 값이 자동으로 넘어갑니다.

로그 확인

엔진 시작/중지, 입력장치 테스트, 픽셀 테스트, 변수 업데이트, 텔레그램 전송 결과 등은 로그에 남습니다. 매크로가 기대대로 움직이지 않으면 먼저 로그를 확인하세요.

13. 키보드/마우스 설정

설정 > 키보드 설정에서 입력 백엔드와 장치를 설정합니다.

항목설명
입력 모드Interception 하드웨어 입력 또는 소프트웨어 입력 방식을 선택합니다.
키보드 장치입력에 사용할 키보드 장치를 선택합니다.
마우스 장치마우스 입력에 사용할 장치를 선택합니다.
테스트 키키보드 테스트에 사용할 키입니다. 기본값은 f24입니다.
키/마우스 딜레이누름-뗌 지연과 입력 사이 지연을 기본값으로 설정합니다.

Interception 드라이버 설치나 장치 선택이 잘못되면 입력이 전송되지 않을 수 있습니다. 설정 후 반드시 테스트 버튼으로 확인하세요.

14. 해상도와 프리셋 이동

픽셀 좌표와 마우스 좌표는 해상도와 앱 배율의 영향을 받습니다. 메인 화면의 해상도 영역은 다른 모니터/배율 환경으로 프리셋을 옮길 때 좌표를 보정하기 위한 기능입니다.

  • 기준 해상도: 프리셋을 만든 환경의 해상도입니다.
  • 기준 앱 배율: 프리셋을 만든 환경의 앱 배율입니다.
  • 현재 해상도: 지금 사용할 화면 해상도입니다. 자동 감지 또는 직접 입력이 가능합니다.
  • 현재 앱 배율: 지금 사용할 앱 배율입니다. 배율 감지 또는 직접 입력이 가능합니다.
  • 기준=현재: 현재 환경을 새 기준으로 저장합니다.
  • 프리셋 옮기기: 프로필의 좌표, 영역, 변수 좌표를 새 환경에 맞게 변환합니다.
좌표 변환 후에는 픽셀 조건과 마우스 클릭 위치를 이미지 뷰어/픽셀 테스트로 다시 확인하는 것이 좋습니다.

15. 예시

예시 1: 트리거를 누르는 동안 키 반복

  1. 매크로 추가에서 트리거를 z, 모드를 홀드로 설정합니다.
  2. 액션에 press r을 추가합니다.
  3. sleep 50ms를 추가합니다.
  4. press t를 추가합니다.
  5. 적용 후 시작/활성 상태에서 z를 누르고 테스트합니다.

예시 2: 특정 색상이 보이면 F1, 아니면 F2

  1. 이미지 뷰어에서 대상 위치에 마우스를 올리고 F1로 좌표를 확인합니다.
  2. F2로 색상을 복사합니다.
  3. 액션 트리에 IF를 추가하고 조건 타입을 픽셀로 설정합니다.
  4. 영역에 x,y,1,1, 색상에 복사한 값을 넣습니다.
  5. THEN에는 press f1, ELSE에는 press f2를 넣습니다.
  6. 픽셀 테스트로 조건이 맞는지 확인한 뒤 적용합니다.

예시 3: 5초마다 텔레그램 알림

  1. 타이머 조건을 만들고 타이머1이 5초 이상이면 참으로 설정합니다.
  2. IF의 THEN에 텔레그램 메시지 액션을 넣습니다.
  3. 그 다음 타이머 액션으로 타이머1을 0초로 초기화합니다.
  4. 매크로가 반복 실행되면 5초마다 메시지가 전송됩니다.

16. 문제 해결

증상확인할 것
트리거가 반응하지 않음엔진 시작 여부, 활성 상태, 매크로 활성 체크, 트리거 키 이름, 앱 범위 설정을 확인합니다.
입력이 전송되지 않음관리자 권한, Interception 설치, 키보드/마우스 장치 선택, 테스트 버튼 결과를 확인합니다.
좌표가 맞지 않음현재 해상도/배율, 기준 해상도/배율, 다중 모니터 좌표, 프리셋 이동 여부를 확인합니다.
픽셀 조건이 안 맞음이미지 뷰어에서 좌표와 색상을 다시 찍고, 허용오차와 최소 일치 픽셀 수를 조정합니다.
매크로가 너무 빠름sleep 액션을 추가하거나 입력 딜레이 기본값/오버라이드를 설정합니다.
무한 반복에서 CPU 사용량이 높음while 그룹 내부에 짧은 sleep을 넣습니다.
변수가 해석되지 않음변수 종류가 맞는지, 이름 오타가 없는지, /name 형식이 맞는지 확인합니다.
